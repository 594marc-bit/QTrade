## Context

QTrade 已有完整的数据采集、因子计算、IC 分析管道。当前因子评分系统可以按日对股票排名并选股，但缺少回测验证环节。系统数据存储在 SQLite 中（~235k 行日线数据），因子通过注册模式管理，评分体系为加权 Z-score。

## Goals / Non-Goals

**Goals:**
- 构建向量化回测引擎，基于现有因子评分系统模拟历史交易
- 支持定期调仓策略（每日/每周/每月），按 total_score 排名选股
- 计算标准回测指标：年化收益、夏普比率、最大回撤、Calmar 比率
- 支持交易成本建模（佣金 + 印花税）
- 生成回测可视化（净值曲线、回撤图、持仓统计）

**Non-Goals:**
- 不做分钟级/ tick 级回测（仅日线）
- 不做做空机制（仅多头）
- 不做机器学习选股
- 不做实时交易执行

## Decisions

### 1. 向量化回测 vs 事件驱动回测

**选择：向量化回测**

利用 pandas 向量化操作按调仓日批量计算持仓和收益，避免逐日逐笔事件循环。适合日线频率的多因子选股策略。

理由：
- 实现简洁，与现有 DataFrame 管道天然契合
- 日线调仓场景下向量化性能远优于事件循环
- 300 只股票 × ~800 交易日，向量化几秒内完成

### 2. 模块结构

```
src/backtest/
  engine.py      # 回测引擎主入口
  portfolio.py   # 组合管理（持仓、调仓）
  metrics.py     # 绩效指标计算
```

- `engine.py`: `BacktestEngine` 类，接收策略参数，驱动整个回测流程
- `portfolio.py`: `Portfolio` 类，管理持仓权重、现金、调仓逻辑
- `metrics.py`: 纯函数，从净值序列计算各项指标

### 3. 调仓逻辑

每个调仓日：
1. 获取当日所有股票的 total_score
2. 排除停牌股（is_trading == False）
3. 选取 top_n 股票
4. 等权分配资金
5. 计算调仓成本（卖出旧持仓 + 买入新持仓）
6. 更新组合净值

调仓频率通过参数控制，用 pandas 的日期重采样确定调仓日。

### 4. 交易成本模型

- 买入佣金：0.03%（可配置）
- 卖出佣金：0.03% + 印花税 0.05%（可配置）
- 滑点：暂不建模，留作后续扩展
- 单次调仓总成本约 0.14%（双向）

### 5. 绩效指标

核心指标：
- 总收益率、年化收益率
- 夏普比率（无风险利率默认 2%）
- 最大回撤、Calmar 比率
- 胜率（日收益 > 0 的比例）
- 调仓次数、平均持仓天数
- 基准对比（沪深300指数）

### 6. 可视化

新增 `src/visualization/backtest_charts.py`：
- 净值曲线（策略 vs 基准）
- 回撤图
- 持仓数量统计
- 年度/月度收益热力图

## Risks / Trade-offs

- **向量化回测的局限**: 无法精确模拟日内价格变动和限价单，对日线策略影响不大 → 后续如需精细模拟可扩展事件驱动模式
- **等权分配假设**: 忽略了实际交易中的最小手数限制（100股） → 对大资金影响小，小资金可能产生偏差
- **停牌处理**: 停牌股票无法卖出，可能锁住资金 → 在调仓时排除停牌股即可避免新买入，已有持仓如遇停牌则被动持有
- **基准数据**: 需要获取沪深300指数日线数据作为基准 → 复用现有 Tushare/AKShare 接口
