## Context

本项目是一个全新的 A 股量化交易系统，从零开始构建。当前项目目录下仅有指南文档，无任何代码。系统聚焦沪深300成分股的动量策略，第一步需要搭建数据管道和因子计算引擎。

## Goals / Non-Goals

**Goals:**
- 建立可靠的数据获取管道，能获取沪深300成分股3年日线行情
- 实现完整的数据清洗流程（缺失值、复权、停牌、异常值、对齐）
- 实现至少3个因子的计算与标准化（动量、成交额动量、波动率）
- 实现因子综合打分和选股排序
- 实现 IC 分析验证因子有效性

**Non-Goals:**
- 不做回测引擎（属于后续阶段）
- 不做风险管理模块（属于后续阶段）
- 不做实盘交易接口（属于后续阶段）
- 不做实时数据推送（仅用日线级别历史数据）
- 不做 Web UI 或可视化（仅 CLI /脚本）

## Decisions

### 1. 数据源：AKShare 优先，Tushare 备选

**选择**: 使用 AKShare 作为主数据源
**原因**: AKShare 免费开源，无需注册 Token，接口稳定，社区活跃。Tushare 需要积分和 Token，对新人有门槛。
**备选**: 如果 AKShare 接口不稳定，可切换到 Tushare，通过配置文件切换。

### 2. 数据存储：SQLite + CSV 双模式

**选择**: 默认用 SQLite 存储，同时支持导出 CSV
**原因**: SQLite 单文件、零配置、支持 SQL 查询，适合个人量化项目的数据量级（~30万行）。CSV 作为交换格式方便调试和分享。
**备选**: 未来数据量增大时可迁移到 PostgreSQL。

### 3. 项目结构：按功能模块组织

```
QTrade/
├── src/
│   ├── data/
│   │   ├── fetcher.py      # 数据获取
│   │   ├── cleaner.py      # 数据清洗
│   │   └── storage.py      # 数据存储
│   ├── factors/
│   │   ├── base.py         # 因子基类
│   │   ├── momentum.py     # 动量因子
│   │   ├── volume.py       # 成交额动量因子
│   │   ├── volatility.py   # 波动率因子
│   │   ├── scorer.py       # 综合打分
│   │   └── ic_analyzer.py  # IC 分析
│   └── config.py           # 配置管理
├── tests/
├── data/                   # 数据文件目录
└── main.py                 # 入口脚本
```

**原因**: 按业务领域（data / factors）划分，每个模块职责清晰，方便后续扩展新因子和新数据源。

### 4. 复权方式：前复权（qfq）

**选择**: 默认使用前复权价格
**原因**: 前复权保持当前价格不变、调整历史价格，更直观地反映真实盈亏。大多数回测系统使用前复权。

### 5. 因子标准化：Z-Score

**选择**: 横截面 Z-Score 标准化
**原因**: 不同因子量纲不同，Z-Score 将因子值标准化到均值0、标准差1，便于加权组合。按交易日做横截面标准化（每天对全市场股票排名）。

## Risks / Trade-offs

- **API 限频风险** → AKShare 对频繁调用有限制，通过增加请求间隔（0.5s）和本地缓存缓解
- **数据质量风险** → 添加数据验证层（价格合理性检查、成交量检查、与公开数据源交叉验证）
- **因子过拟合风险** → IC 分析要求在足够长的时间窗口（至少1年）上评估，避免短期巧合
- **未来函数风险** → 因子计算严格使用 `shift()` 确保只用历史数据，IC 分析中未来收益用 `shift(-5)` 显式标注
